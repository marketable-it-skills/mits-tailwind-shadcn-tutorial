# Module 2: Playing with Colors - Light & Dark Cards

## Overview

In this module, you'll explore Tailwind CSS's powerful color system by creating three different card variations: a light theme card, a dark theme card, and a colored accent card. This hands-on exploration will help you understand how colors, contrast, and design tokens work together to create beautiful interfaces.

## What You'll Learn

- **Tailwind's color system** - Understanding color scales (50, 100, 200...900, 950)
- **Color naming conventions** - How gray, slate, blue, and other colors are organized
- **Text contrast and readability** - Choosing the right text colors for backgrounds
- **Color relationships** - How background, text, and border colors work together
- **Responsive layout** - Displaying multiple cards side-by-side

## Why This Matters

Understanding color systems is crucial for:

1. **Creating consistent designs** - Using a predefined palette ensures harmony
2. **Building theme support** - Light/dark modes require understanding color contrast
3. **Accessibility** - Proper contrast ratios make your UI usable for everyone
4. **Design confidence** - No more guessing hex codes or RGB values

Before building complex theme systems (Module 10), you need to understand how Tailwind's colors work at a fundamental level.

## What We're Building

Three card variations displayed side-by-side (or stacked on mobile):

1. **Light Card** - White background, dark text, light borders
2. **Dark Card** - Dark slate background, light text, dark borders
3. **Colored Card** - Blue tinted background, blue text, blue borders

Each card will display the same content but with different color schemes, helping you visualize how color choices affect the overall design.
